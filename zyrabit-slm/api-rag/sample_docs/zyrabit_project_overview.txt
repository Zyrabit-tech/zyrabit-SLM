ZYRABIT SLM SECURE SUITE - PROJECT OVERVIEW

Zyrabit SLM Secure Suite is a local AI solution that combines Small Language Models (SLMs) with a RAG engine and a Zero-Trust security layer.

PHILOSOPHY
- Efficiency: Optimized execution for consumer hardware (Mac M1/M2, consumer GPUs)
- Speed: Low latency with compact models
- Sovereignty: Your data never leaves your infrastructure

ARCHITECTURE
The solution is organized in layers:
1. Edge/Ingress Layer: Traefik receives traffic and applies TLS, redirects and rate limiting
2. Application Layer: api-rag exposes HTTP endpoints and coordinates logic
3. Security Layer: Pipeline for anonymization/deanonymization of sensitive data (PII)
4. Inference Layer: slm-engine (Ollama) runs the local model
5. Knowledge Layer: vector-db (ChromaDB) stores and retrieves embeddings/context
6. Observability Layer: Prometheus, Grafana and optionally Loki

SERVICES
- traefik: Ingress, HTTPS and entry policies
- api-rag: FastAPI core
- slm-engine: Local inference engine (Ollama)
- vector-db: ChromaDB for semantic persistence
- prometheus + grafana: Monitoring
- n8n (optional): Automation via webhook

RAG AND INGESTION
- Documents can be ingested via POST /v1/ingest (PDF, TXT, MD)
- The ingest script loads from document_source directory
- ChromaDB stores embeddings generated by Ollama (mxbai-embed-large)
- The router decides between RAG (search_rag_database) or direct SLM (direct_SLM_answer)
- Keywords that trigger RAG: zyrabit, architecture, security, slm, rag, chromadb, ollama, docker

SECURITY
- PII anonymization before inference
- Reversible deanonymization of tokens when responding
- Zero-Trust local topology: no sensitive data leaves the infrastructure
- Network segmentation: frontend-network, backend-network, model-network (internal)

API ENDPOINTS
- GET /health: Health check
- POST /v1/chat: Main router for queries (RAG or direct SLM)
- POST /v1/ingest: Ingest documents into knowledge base
- GET /metrics: Prometheus metrics
- POST /mcp: MCP JSON-RPC
- POST /v1/integrations/n8n/webhook: n8n webhook integration
